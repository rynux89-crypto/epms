# UPS 모니터링 시스템 구축 제안서

문서 버전: 1.0  
작성일: 2026-07-20  
대상 시스템: EPMS 연계형 UPS 모니터링 시스템

## 1. 제안 개요

본 제안서는 UPS 설비의 운전 상태, 전력 품질, 알람, 이벤트, 측정 이력을 통합적으로 감시하기 위한 UPS 모니터링 시스템 구축 방안을 제시한다.

제안 시스템은 Modbus TCP 기반 UPS 장비를 대상으로 실시간 데이터를 수집하고, 운영자가 웹 화면에서 UPS 상태를 즉시 확인할 수 있도록 구성된다. 단독 UPS 모니터링 시스템으로도 사용할 수 있으며, 기존 EPMS 시스템의 하위 모듈로도 연계 가능하도록 설계한다.

## 2. 제안 배경

UPS는 전산실, 전기실, 생산설비, 통신설비 등 중요 부하의 전원 연속성을 보장하는 핵심 설비이다. 그러나 UPS 상태를 현장 장비 패널 또는 수동 점검에 의존할 경우 다음과 같은 운영 문제가 발생할 수 있다.

- UPS 장애 또는 경고 발생 시 인지 지연
- 배터리 저하, 과부하, 입력 이상 등 위험 징후의 사전 파악 어려움
- 여러 대의 UPS를 통합적으로 비교하기 어려움
- 알람, 이벤트, 측정 이력의 체계적 관리 부족
- 보고서 작성 및 장애 분석에 필요한 데이터 확보 어려움

본 시스템은 이러한 문제를 해결하기 위해 UPS 운전 정보를 자동 수집하고, 대시보드와 상세 화면을 통해 운영자가 한 화면에서 전체 상태를 판단할 수 있도록 지원한다.

## 3. 제안 목표

본 사업의 주요 목표는 다음과 같다.

- UPS 통합 감시 체계 구축
- UPS별 실시간 운전 상태 및 전력 흐름 시각화
- 알람 및 이벤트 이력 관리
- 측정 데이터 저장 및 이력 조회
- 기간별 보고서 및 CSV 다운로드 제공
- UPS 등록, 수정, 삭제, 활성 여부 관리
- UPS별 데이터 수집주기 관리
- 향후 라이선스 기반 장비 수량 관리 체계 확장
- EPMS 메인 시스템과 독립 UPS 모듈 양쪽에서 사용 가능한 구조 확보

## 4. 제안 시스템 범위

### 4.1 구축 대상

구축 대상은 Modbus TCP 통신을 지원하는 3상 UPS 설비이다. 현재 시스템은 Schneider Easy UPS 3-Phase Modular 프로파일을 기준으로 구현되어 있으며, 향후 UPS 제조사 또는 모델별 레지스터 맵을 추가하여 확장할 수 있다.

### 4.2 주요 제공 화면

| 화면 | 주요 기능 |
| --- | --- |
| UPS 메인 | UPS 관련 화면 통합 메뉴 |
| UPS 통합 대시보드 | 전체 상태, KPI, UPS 개요, 전력 흐름, 추이, 배치도, 알람, 이벤트, 장비 현황 |
| UPS 전체 현황 | 등록 UPS 타일/리스트 조회, 활성/비활성 UPS 확인 |
| UPS 모니터링 | 선택 UPS의 상세 실시간 상태 및 미믹 다이어그램 |
| Phasor Diagram | 전압/전류 위상 벡터 및 파형 시각화 |
| UPS 알람 | 활성 알람 및 과거 알람 조회, CSV 다운로드 |
| UPS 이벤트 | 운전 상태 변화, 차단기 상태 변화, 시나리오 변화 등 이벤트 조회 |
| UPS 측정 이력 | 수집된 측정값 이력 조회 |
| UPS 레포트 | 기간별 통계 보고서 및 CSV 다운로드 |
| UPS 등록 | UPS 장비 정보, 용량, 위치, 수집주기, 활성 여부 관리 |
| UPS 초기 설정 | DB 및 수집기 상태 확인, 이력 데이터 관리 |
| UPS 시뮬레이터 | 테스트용 UPS 데이터 및 알람/차단기 시나리오 발생 |

## 5. 주요 기능 제안

### 5.1 통합 대시보드

통합 대시보드는 운영자가 가장 먼저 확인하는 메인 화면이다.

제공 기능:

- 전체 상태 카드
- 전체 부하율 카드
- 총 출력 전력 카드
- 배터리 평균 카드
- 경고 알람 카드
- UPS 개요 카드
- 전력 흐름 카드
- 실시간 추이 카드
- UPS 건강 상태 카드
- UPS 배치도 카드
- 최근 알람 및 최근 이벤트 카드
- 장비 현황 카드

운영 기준:

- 최초 로딩 시 정상 상태의 활성 UPS를 우선 선택
- 비활성 UPS는 기본 선택 목록에서 제외
- 오프라인 UPS는 전압, 주파수, 운전모드, 전력 흐름을 실제 운전값처럼 표시하지 않음
- 실시간 추이는 DB에 저장된 측정 데이터를 분 단위 60개 구간으로 집계하여 표시

### 5.2 UPS 상세 모니터링

UPS 상세 모니터링 화면은 선택 UPS의 현재 상태를 상세히 확인하는 화면이다.

표시 항목:

- UPS 모드
- 시스템 모드
- 입력/출력 전압
- 출력 전류
- 출력 전력
- 피상전력
- 주파수
- 부하율
- 배터리 잔량
- 배터리 전류
- 차단기 상태
- 미믹 다이어그램 기반 전력 흐름

미믹 다이어그램은 UIB, UOB, SSIB, BF2, MBB, BB 상태에 따라 상용전원, UPS, 부하, 배터리 사이의 전력 경로를 시각적으로 표현한다.

### 5.3 Phasor Diagram

Phasor Diagram 화면은 3상 전압 및 전류의 위상 관계를 시각적으로 표시한다.

제공 기능:

- 전압 원형 벡터 그래프
- 전류 원형 벡터 그래프
- 3상 전압 파형
- 3상 전류 파형
- L1/R상, L2/S상, L3/T상 구분
- 역률 기반 전류 위상 추정

현재 레지스터에 leading/lagging 방향 정보가 없는 경우, 전류 위상은 지상 부하 기준으로 추정한다. 향후 정확한 진상/지상 판단을 위해서는 UPS 레지스터 맵에 PF 방향 정보 또는 kVAR 부호 정보가 필요하다.

### 5.4 알람 관리

UPS 알람 화면은 현재 활성 알람과 과거 알람을 조회한다.

대상 알람 예:

- UPS 통신 실패
- 배터리 잔량 낮음
- 배터리 잔량 위험
- 출력 부하율 높음
- 출력 부하율 위험
- 입력 이상
- 출력 이상
- 바이패스 이상
- 파워 모듈 이상
- 중요 알람

알람은 warning, critical 등급으로 구분하여 화면 색상과 배치도 상태 색상에 반영한다.

### 5.5 이벤트 관리

이벤트는 UPS 운영 상태 변화 또는 운전 이력 분석에 필요한 상태 변화를 기록한다.

대상 이벤트 예:

- UPS 운전모드 변경
- 시스템 모드 변경
- 입력 이상 발생/복구
- 출력 이상 발생/복구
- 배터리 운전 시작/종료
- 바이패스 운전 시작/종료
- EPO 동작
- 차단기 상태 변경
- 시뮬레이터 시나리오 변경

알람이 즉각적인 조치 대상이라면, 이벤트는 운전 상태의 변화 이력을 남기는 목적이다.

### 5.6 측정 이력 및 보고서

수집된 UPS 데이터는 DB에 저장되며, 이력 조회와 보고서 화면에서 활용한다.

제공 기능:

- 기간별 측정 데이터 조회
- UPS별 검색
- 알람/이벤트 이력 조회
- 평균/최대/최소 데이터 확인
- CSV 다운로드
- 한글 CSV 깨짐 방지를 위한 UTF-8 BOM 처리

### 5.7 UPS 등록 및 수집주기 관리

UPS 등록 화면에서는 장비별 기본 정보를 관리한다.

등록 항목:

- UPS 이름
- 위치
- IP 주소
- Port
- Unit ID
- 정격 용량(kVA)
- 수집주기(초)
- Modbus 프로파일
- 활성 여부

수집기는 UPS별 수집주기에 따라 측정값을 수집한다. 알람성 데이터는 운영 요구에 따라 더 빠른 주기로 확인하도록 개선 가능하다.

### 5.8 시뮬레이터

시뮬레이터는 실제 UPS 없이도 화면, 알람, 이벤트, 차단기 상태, 전력 흐름을 테스트할 수 있도록 제공한다.

지원 시나리오 예:

- 정상
- 배터리 운전
- 배터리 부족
- 과부하
- 입력 이상
- 출력 이상
- 바이패스 운전
- 바이패스 이상
- 파워 모듈 이상
- 중요 알람
- EPO 동작

## 6. 시스템 구성

### 6.1 구성 개요

```text
UPS 장비
  ↓ Modbus TCP
UPS 수집기
  ↓
UPS_MONITOR DB
  ↓
Tomcat / JSP Web Application
  ↓
운영자 웹 화면 / EPMS 메인 연계
```

### 6.2 주요 구성 요소

| 구성 요소 | 설명 |
| --- | --- |
| UPS 장비 | Modbus TCP 통신 대상 설비 |
| UPS 수집기 | UPS별 수집주기에 따라 레지스터 값을 읽고 DB 저장 |
| UPS DB | 장비 정보, 측정값, 알람, 이벤트, 설정 정보 저장 |
| Web Application | JSP 기반 운영 화면 제공 |
| EPMS 연계 메뉴 | EPMS 메인 화면에서 UPS 화면으로 이동 |
| 시뮬레이터 | 개발/검증용 Modbus TCP UPS 에뮬레이터 |

### 6.3 DBMS

현재 기본 구성은 Microsoft SQL Server 기반이다. DB 접속 정보는 UPS 애플리케이션 설정 파일 또는 Tomcat JNDI 설정을 통해 관리한다.

향후 다른 DBMS로 전환하려면 다음 항목의 검토가 필요하다.

- JDBC Driver 교체
- SQL 문법 차이 검토
- IDENTITY, 날짜 함수, 페이징 문법 차이 반영
- 스키마 생성 스크립트 분리
- DB 접속 설정 외부화

## 7. 운영 및 보안 제안

### 7.1 운영 기준

- UPS별 수집주기는 설비 중요도와 UPS 수량에 따라 설정
- 알람성 데이터는 실시간성을 우선하여 별도 빠른 확인 주기 적용 권장
- 비활성 UPS는 수집 대상과 드롭다운 선택 목록에서 제외
- 전체 현황에서는 필요 시 비활성 UPS도 조회 가능
- 오프라인 UPS의 과거 측정값은 현재값처럼 표시하지 않음

### 7.2 보안 및 권한

향후 운영 환경에서는 다음 기능을 추가하는 것이 권장된다.

- 사용자 로그인 및 권한 분리
- 관리자 전용 UPS 등록/삭제 권한
- 알람 확인 처리 권한
- 설정 변경 이력 기록
- DB 접속정보 암호화 또는 외부 설정 관리
- 라이선스 파일 서명 검증

## 8. 라이선스 운영 방안

본 시스템은 향후 UPS 수량 기반 라이선스 체계로 확장할 수 있다.

권장 방식:

- 고객사별 라이선스 파일 발급
- 허용 UPS 수량, 만료일, 고객명, 시스템 식별값 포함
- 오프라인 환경에서도 검증 가능한 전자서명 방식 적용
- 서버 실행 시 라이선스 검증
- UPS 등록 또는 활성화 시 허용 수량 초과 여부 확인

라이선스 발급 프로그램은 별도 관리자용 도구로 구성하는 것이 적절하다. 운영망이 오프라인인 경우를 고려하여, 온라인 인증 서버 의존 방식보다는 서명된 라이선스 파일 방식이 적합하다.

## 9. 기대 효과

본 시스템 도입 시 기대 효과는 다음과 같다.

- 여러 대의 UPS 상태를 한 화면에서 통합 감시
- UPS 장애 및 경고 상황의 조기 인지
- 배터리 저하, 과부하, 입력/출력 이상 등 위험 요소의 시각적 확인
- 알람/이벤트 이력 기반 장애 원인 분석 가능
- 측정 이력 및 보고서 기반 운영 데이터 축적
- EPMS와 UPS 화면의 통합 운영으로 관리 효율 향상
- 시뮬레이터 기반 사전 검증으로 운영 리스크 감소
- 향후 장비 수량 기반 라이선스 사업 모델 적용 가능

## 10. 구축 일정안

| 단계 | 기간 | 주요 내용 |
| --- | --- | --- |
| 1단계: 요구사항 확정 | 1주 | UPS 모델, 레지스터 맵, 화면 범위, 알람 기준 확정 |
| 2단계: 기본 구축 | 2주 | DB 구성, UPS 등록, 수집기, 기본 화면 구축 |
| 3단계: 대시보드/상세화면 구축 | 2주 | 통합 대시보드, 모니터링, Phasor, 배치도, 전력 흐름 구현 |
| 4단계: 알람/이벤트/보고서 구축 | 2주 | 알람 기준, 이벤트 기준, 이력 조회, CSV 보고서 구현 |
| 5단계: 테스트 및 안정화 | 1주 | 시뮬레이터 테스트, 실제 UPS 연동 테스트, 화면 검증 |
| 6단계: 교육 및 인수 | 1주 | 운영자 교육, 운영매뉴얼 제공, 인수 테스트 |

총 예상 기간은 약 9주이며, 기존 구현 범위를 활용하는 경우 실제 구축 기간은 단축될 수 있다.

## 12. 결론

UPS 모니터링 시스템은 중요 전원 설비의 상태를 실시간으로 파악하고, 장애 발생 전후의 데이터를 체계적으로 관리하기 위한 운영 시스템이다.

본 제안 시스템은 대시보드, 상세 모니터링, Phasor Diagram, 알람, 이벤트, 이력, 보고서, 시뮬레이터 기능을 포함하고 있으며, EPMS 전체 시스템의 일부로도, 독립 UPS 모니터링 시스템으로도 운영 가능하다.

이를 통해 운영자는 UPS 상태를 빠르게 판단하고, 장애 대응 시간을 줄이며, 전원 설비 운영의 신뢰성과 가시성을 높일 수 있다.

# UPS 모니터링 시스템 운영매뉴얼

문서 버전: 1.0  
작성일: 2026-06-16  
대상 시스템: EPMS UPS 모니터링 시스템

## 1. 개요

UPS 모니터링 시스템은 Modbus TCP 기반 UPS 장비를 등록하고, 수집된 운전 데이터를 대시보드, 실시간 상태, Phasor Diagram, 알람, 이벤트, 이력, 보고서 화면에서 확인하는 운영 시스템이다.

주요 기능은 다음과 같다.

- UPS 등록, 수정, 삭제 및 활성 여부 관리
- UPS별 수집주기 설정
- UPS 통합 대시보드 조회
- 실시간 UPS 상태 및 전력 흐름 확인
- Phasor Diagram 및 파형 확인
- 알람 및 이벤트 이력 조회
- 측정 이력 조회 및 CSV 다운로드
- 기간별 UPS 보고서 조회 및 CSV 다운로드
- UPS 시뮬레이터 상태 확인
- UPS DB 및 수집기 상태 점검

## 3. 운영 메뉴

### 3.1 UPS 메인

UPS 관련 전체 메뉴로 이동하는 시작 화면이다.

운영자는 다음 화면으로 이동할 수 있다.

- UPS 통합 대시보드
- UPS 전체 현황
- UPS 모니터링
- Phasor Diagram
- UPS 레포트
- 측정 이력
- 알람
- 이벤트
- UPS 등록
- UPS 초기 설정
- UPS 시뮬레이터

### 3.2 UPS 통합 대시보드

운영 상황을 한 화면에서 확인하는 메인 대시보드이다.

주요 카드:

- 전체 상태
- 전체 부하율
- 총 출력 전력
- 배터리 평균
- 경고 알림
- UPS 개요
- 전력 흐름
- 실시간 추이
- UPS 건강 상태
- UPS 배치도
- 경고 알림 리스트
- 장비 현황 리스트

운영 기준:

- 최초 로딩 시 정상 상태의 활성 UPS를 우선 선택한다.
- UPS 선택 드롭다운에는 활성화된 UPS만 표시한다.
- 비활성 UPS는 대시보드 선택 대상에서 제외한다.
- 오프라인 UPS는 전압, 운전모드, 전력 흐름, 건강상태 데이터를 실제 값처럼 표시하지 않고 `--` 또는 오프라인 상태로 표시한다.

### 3.3 UPS 전체 현황

등록된 UPS 목록을 타일 또는 리스트 형태로 확인한다.

운영 기준:

- 기본 조회는 활성 UPS 중심이다.
- `비활성 포함` 옵션을 사용하면 비활성 UPS도 함께 확인할 수 있다.
- 검색창에서 UPS 이름 또는 위치로 필터링할 수 있다.
- 상태, 위치, 최근 수집, 부하율, 출력, 배터리, 알람 수를 확인한다.

### 3.4 UPS 모니터링

선택한 UPS의 상세 실시간 상태를 확인한다.

표시 항목:

- UPS 모드
- 시스템 모드
- 전압
- 전류
- 주파수
- 역률
- 총 부하
- 배터리 상태
- UIB, SSIB, UOB, BF2, MBB, BB 상태
- 미믹 다이어그램 전력 경로

운영 기준:

- UPS 선택 드롭다운에는 활성 UPS만 표시한다.
- 비활성 UPS ID로 직접 접근하면 첫 번째 활성 UPS로 자동 이동한다.
- 활성 UPS가 없으면 기존 비활성 UPS의 데이터가 남아 보이지 않도록 빈 상태로 표시한다.
- 통신 불량 UPS는 운전모드와 데이터 표시를 제한한다.

### 3.5 UPS Phasor Diagram

출력 전압과 전류의 위상 관계를 시각적으로 확인한다.

표시 항목:

- 전압 Phasor 원형 그래프
- 전류 Phasor 원형 그래프
- 전압 파형 그래프
- 전류 파형 그래프
- Y축 스케일 값
- 최근 수집 시간

운영 기준:

- UPS 선택 드롭다운에는 활성 UPS만 표시한다.
- 비활성 UPS ID로 직접 접근하면 첫 번째 활성 UPS로 자동 이동한다.
- 역률은 `output_pf_l1`, `output_pf_l2`, `output_pf_l3` 값을 사용한다.
- 현재 레지스터에는 leading/lagging 방향 정보가 없으므로, 전류는 기본적으로 지상 부하, 즉 lagging으로 가정한다.
- 정확한 진상/지상 판별이 필요하면 PF 방향 레지스터, kVAR 부호, 또는 별도 방향 컬럼이 필요하다.

## 4. UPS 등록 및 관리

### 4.1 UPS 등록

경로: `/ups/system/ups_register.jsp`

입력 항목:

- UPS 이름
- 위치
- IP 주소
- Port
- Unit ID
- 용량(kVA)
- 수집주기(초)
- 프로파일
- 활성 여부

운영 절차:

1. UPS 등록 화면으로 이동한다.
2. UPS 이름, 위치, IP, Port, Unit ID를 입력한다.
3. 용량(kVA)을 입력한다.
4. 수집주기(초)를 입력한다. 기본값은 2초이다.
5. 프로파일을 선택한다.
6. 수집 대상이면 활성 체크박스를 선택한다.
7. `등록` 버튼을 클릭한다.

### 4.2 UPS 수정

운영 절차:

1. 등록된 UPS 리스트에서 `수정` 버튼을 클릭한다.
2. 상단 입력폼에 기존 UPS 정보가 표시된다.
3. 필요한 값을 수정한다.
4. `수정 저장` 버튼을 클릭한다.

### 4.3 UPS 비활성 처리

수집 대상에서 제외하려면 해당 UPS의 `활성` 체크를 해제한다.

비활성 UPS 처리 기준:

- 수집 대상에서 제외된다.
- 대시보드, UPS 모니터링, Phasor 화면의 선택 드롭다운에 표시되지 않는다.
- 전체 현황 화면에서는 `비활성 포함` 옵션으로 조회할 수 있다.
- 등록 리스트에는 활성/비활성 여부와 관계없이 표시된다.

### 4.4 UPS 삭제

운영 절차:

1. 등록된 UPS 리스트에서 `삭제` 버튼을 클릭한다.
2. 확인 메시지를 검토한다.
3. 확인하면 UPS 정보와 관련 측정/알람 이력이 삭제된다.

주의:

- 삭제된 UPS ID는 재사용되지 않는다.
- SQL Server IDENTITY 컬럼 특성상 신규 UPS는 새로운 ID를 부여받는다.
- 삭제 전 이력 보존이 필요하면 측정 이력 또는 보고서를 CSV로 다운로드한다.

## 5. 데이터 수집

### 5.1 수집 구조

UPS 수집기는 등록된 활성 UPS를 대상으로 Modbus TCP 통신을 수행한다.

기본 동작:

- UPS별 `poll_interval_seconds` 값에 따라 수집한다.
- 수집 성공 시 `ups_measurement` 테이블에 측정값을 저장한다.
- 통신 성공/실패 상태는 UPS 장비 정보에 반영한다.
- 알람 조건에 해당하면 알람 이력을 생성한다.

### 5.2 수집주기

수집주기는 UPS 등록 화면에서 초 단위로 입력한다.

운영 기준:

- 일반적인 테스트 환경은 2초 수집을 사용한다.
- 운영 환경에서는 DB 부하, UPS 수량, 네트워크 상태를 고려해 수집주기를 조정한다.
- 너무 짧은 수집주기는 DB 저장량과 화면 조회 부하를 증가시킨다.

### 5.3 실시간 추이 데이터

대시보드 실시간 추이는 DB에 저장된 측정값을 기준으로 표시한다.

현재 표시 방식:

- 최근 데이터를 분 단위 구간으로 나눈다.
- 최대 60개 구간을 집계하여 그래프에 표시한다.
- 오프라인 UPS는 실시간 추이 데이터를 표시하지 않는다.

## 9. DB 접속 정보

UPS DB 접속 정보는 다음 우선순위로 사용한다.

1. JNDI DataSource: `java:comp/env/jdbc/ups`
2. 환경변수
3. `WEB-INF/config.toml`의 `[ups_database]` 설정

환경변수 예:

```text
UPS_DB_SERVER
UPS_DB_NAME
UPS_DB_USER
UPS_DB_PASSWORD
UPS_DB_ENCRYPT
UPS_DB_TRUST_SERVER_CERTIFICATE
```

운영 원칙:

- 운영 서버에서는 비밀번호를 JSP 소스에 직접 넣지 않는다.
- 가능하면 Tomcat JNDI 또는 환경변수로 관리한다.
- DBMS 변경이 필요한 경우 JDBC 드라이버, DataSource 설정, SQL 문법 호환성을 함께 검토한다.

## 10. 시뮬레이터 운영

경로: `/ups/simulator/index.jsp`

시뮬레이터는 별도 Python 프로세스로 실행된다.

기본 실행 예:

```powershell
cd "C:\Tomcat 9.0\webapps\ROOT\ups\simulator"
python ups_modbus_simulator.py --host 127.0.0.1 --port 1502 --control-port 1503 --scenario normal --open-browser
```

등록 예:

- IP: `127.0.0.1`
- Port: `1502`
- Unit ID: `1`
- 프로파일: `Schneider Easy UPS 3-Phase Modular`

확인:

- 시뮬레이터 UI: `http://127.0.0.1:1503/`
- UPS 시뮬레이터 화면에서 실행 여부를 확인한다.

시뮬레이터에서 조작 가능한 주요 값:

- 현재 값 필드
- 부하율
- 배터리 잔량
- 배터리 잔여시간
- 역률
- UIB, SSIB, UOB, BF2, MBB, BB 상태
- 정상, 배터리 운전, 저배터리 등 시나리오

## 11. 배포 및 반영

소스 위치:

- 개발/소스 기준: `C:\Tomcat 9.0\webapps\ROOT\ups`
- 운영 배포 기준: `C:\Tomcat 9.0\webapps\ups`

운영 반영 절차:

1. `ROOT\ups`에서 소스를 수정한다.
2. 수정 파일을 `webapps\ups` 운영 폴더로 복사한다.
3. Java 파일 수정 시 컴파일 후 class 파일을 운영 폴더에 복사한다.
4. `ups.war`, `ups.deploy.war`를 재생성한다.
5. 브라우저에서 `Ctrl + F5`로 새로고침한다.
6. 필요 시 Tomcat을 재시작한다.

WAR 재생성 예:

```powershell
cd "C:\Tomcat 9.0\webapps\ROOT\ups"
jar -cf "C:\Tomcat 9.0\webapps\ups.war" *
jar -cf "C:\Tomcat 9.0\webapps\ups.deploy.war" *
```

JSP만 수정한 경우:

- 운영 폴더에 JSP를 복사하면 즉시 반영되는 경우가 많다.
- 단, include 파일, class 변경, 캐시 영향이 있으면 Tomcat 재시작이 필요할 수 있다.

Java class 수정 시:

- `javac`로 컴파일한다.
- `WEB-INF/classes/epms/ups` 하위 class 파일을 운영 폴더에도 복사한다.
- 이미 로드된 class가 계속 사용되면 Tomcat 재시작이 필요하다.

## 12. 장애 대응

### 12.1 화면에 UPS가 보이지 않음

확인 순서:

1. UPS 등록 화면에서 해당 UPS가 등록되어 있는지 확인한다.
2. 활성 체크 여부를 확인한다.
3. 대시보드, UPS 모니터링, Phasor 화면은 활성 UPS만 표시한다.
4. 전체 현황에서 `비활성 포함` 옵션으로 확인한다.
5. DB 연결 오류가 있는지 초기 설정 화면에서 확인한다.

### 12.2 데이터가 업데이트되지 않음

확인 순서:

1. UPS 등록 화면에서 수집주기를 확인한다.
2. UPS 초기 설정 화면에서 수집기 상태를 확인한다.
3. 최근 수집 시간이 갱신되는지 확인한다.
4. UPS IP, Port, Unit ID가 맞는지 확인한다.
5. 방화벽 또는 네트워크 연결을 확인한다.
6. 시뮬레이터 장비이면 Python 프로세스 실행 여부를 확인한다.

### 12.3 오프라인인데 값이 보임

정상 기준:

- 오프라인 UPS는 주요 전압, 운전모드, 전력 흐름, 건강상태를 실제 운전값처럼 표시하지 않아야 한다.
- 오래된 DB 값이 남아 있더라도 현재 통신 상태가 오프라인이면 화면에서 제한 표시한다.

확인 순서:

1. 최근 수정 파일이 운영 폴더에 복사되었는지 확인한다.
2. `ups.war`, `ups.deploy.war`가 최신인지 확인한다.
3. 브라우저 캐시를 삭제하거나 `Ctrl + F5`를 실행한다.
4. Java class 변경이 포함되었으면 Tomcat을 재시작한다.

### 12.4 알람 카드에는 건수가 있는데 알람 화면에 안 보임

확인 순서:

1. 대시보드 알람 링크가 `/ups/alarm/alarm_view.jsp?status=ACTIVE`인지 확인한다.
2. 알람 화면이 현재 활성 알람을 표시하는지 확인한다.
3. 알람 상태가 `ACTIVE`인지 DB에서 확인한다.
4. 해제된 알람은 일반 이력 조회에서 확인한다.

### 12.5 CSV 한글이 깨짐

확인 순서:

1. CSV 다운로드 기능이 UTF-8 BOM을 포함하는지 확인한다.
2. Excel에서 바로 열어 깨지면 `데이터 > 텍스트/CSV에서` 기능으로 UTF-8을 선택한다.
3. 그래도 깨지면 브라우저 다운로드 파일 인코딩과 응답 헤더를 확인한다.

## 15. 주요 운영 원칙

- 운영 화면에서 UPS 선택 드롭다운은 활성 UPS만 표시한다.
- 비활성 UPS는 수집 및 주요 실시간 화면 선택 대상에서 제외한다.
- 삭제는 이력 삭제를 동반하므로 신중하게 수행한다.
- 수집주기 변경은 DB 부하와 화면 갱신 요구를 함께 고려한다.
- 오프라인 UPS의 과거 측정값을 현재값처럼 표시하지 않는다.
- 배포 후에는 운영 폴더와 WAR 파일이 모두 최신인지 확인한다.
- Java class 변경 후 이상 동작이 있으면 Tomcat 재시작을 우선 고려한다.
